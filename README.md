# bSAM-SVI-SWA

This repository contains an upgraded implementation of Bayesian Sharpness-Aware Minimization with SVI and SWA-style posterior averaging for image classification.

## Theoretical Background

Sharpness-Aware Minimization (SAM) can be interpreted as an approximation to Bayesian learning. In this project framing, bSAM treats SAM as an optimal relaxation of the Bayes objective: instead of directly optimizing the intractable expected negative-loss under a parameter distribution, we optimize its best convex lower bound given by the Fenchel biconjugate. This viewpoint connects robust local geometry (sharpness control) with variational Bayesian inference.

Concretely, the optimization maintains both parameter means and a learned precision/variance structure, allowing the algorithm to adapt uncertainty during training. Because the variance parameter is learned rather than fixed, the resulting model can provide uncertainty-aware predictions through posterior sampling and model averaging at test time.

### Mathematical Formulation of bSAM

The standard SAM objective seeks to minimize the maximum loss in a local $\rho$-neighborhood:
$$\min_{\theta} \max_{||\epsilon|| \le \rho} l(\theta + \epsilon)$$

The bSAM framework connects this to the Bayes objective by replacing the expected negative-loss with its Fenchel biconjugate. This yields a relaxed objective that optimizes the mean $m$ and variance $\sigma^2$ of an isotropic Gaussian posterior:
$$\mathcal{E}_{relaxed}(m, \sigma) = \left[ \sup_{\epsilon} l(m + \epsilon) - \frac{1}{2\sigma^2}||\epsilon||^2 \right] + \frac{\delta'}{2}||m||^2 - \frac{P}{2}\log(\sigma^2 \delta')$$

Here, the hard constraint $\rho$ from standard SAM emerges naturally as a function of the learned variance $\sigma^2$.

## Project Scope and Goals

The implementation focuses on:

- Extending the original bSAM training recipe with an SVI+SWA variant.
- Improving practical training/evaluation behavior for large-scale experimentation.
- Exposing uncertainty quantification via posterior predictive sampling.
- Keeping compatibility with the original JAX/Haiku training stack while adding supplementary PyTorch SAM materials.

## Implementation Overview

Core components of the repository:

- `train.py`: main training pipeline, optimizer setup, schedule, periodic evaluation, checkpoint serialization.
- `optim.py`: optimizer builders for SGD, SAM, bSAM, and the added bSAM-SVI-SWA variant.
- `test.py`: inference and evaluation script with deterministic and Bayesian averaging modes.
- `util.py`: calibration/metrics helpers, likelihood, sampling utilities.
- `models.py`, `resnet18.py`: model definitions.
- `data.py`: dataset loading/preprocessing.
- `torch_optim.py` and `Example_of_use_torch_optim.ipynb`: standalone PyTorch SAM reference implementation and usage example.

## Reproducibility Setup and Commands

The following setup and execution instructions are kept exactly as in the existing README.

This is the repository of bSAM upgrade learning project. To train and test our method, follow these steps:

1) **Set up the environment:** First you need to create Python 3.10 venv on Linux or WSL2 (Windows is not natively supported by JAX):

python3.10 -m venv bsam-env

source bsam-env/bin/activate

2) **Install dependencies:** Then, install the required packages in the following order:

pip install typing-extensions==4.12.2

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128

pip install --upgrade "jax[cuda12]" -f https://storage.googleapis.com/jax-releases/jax_cuda_releases.html

pip install -r requirements.txt

*Note: Ensure CUDA 12.8 is installed if not present.*

3) **Training:** To train with ResNet-18 on CIFAR10 dataset, run:

python train.py --alpha 0.5 --beta1 0.9 --beta2 0.999 --priorprec 40 --rho 0.05 --batchsplit 8 --optim bsam_swi_swa --dataset cifar10 --dafactor 4 --batchsize 128

4) **Testing:** To test a trained model, use:

python test.py --resultsfolder results/cifar10_resnet18/bsam/run_0 --testmc 32

Additionally, a PyTorch implementation of the original SAM optimizer is provided in the following files: Example_of_use_torch_optim.ipynb and torch_optim.py.

## Experimental Workflow

Typical workflow for this repository:

1. Prepare environment and dependencies.
2. Train with the selected optimizer variant (`sgd`, `sam`, `bsam`, or `bsam_svi_swa` in code).
3. Save `trainstate.pickle` checkpoints and run metadata.
4. Evaluate with point estimates and optional Bayesian model averaging.
5. Compare accuracy, NLL, calibration behavior, and uncertainty quality.

## Outputs and Artifacts

Training outputs are organized under `results/` by dataset, architecture, optimizer, and run id. The main artifacts are:

- `trainstate.pickle`: serialized optimizer/model state and arguments.
- `info.txt` (when produced): run configuration summary.
- Additional run-level logs/plots generated by helper scripts.

## Limitations and Notes

- The JAX CUDA setup is targeted at Linux/WSL2 as already noted above.
- The repository contains both JAX and PyTorch components; only selected files are used in each workflow.
- Some command defaults in current scripts are tuned for faster iteration (for example, partial-batch evaluation), so full-benchmark parity may require adjusting those limits.
